## Vault Structure

```
00 Inbox/       — unprocessed capture, route daily
01 Daily/       — session notes and daily logs
02 Context/     — permanent identity files (me.md, strategy, brand)
03 Projects/    — one subfolder per active project
04 Resources/   — personal library: prompts, frameworks, references
prompts/        — ready-to-use prompt templates
```

## Core Rules

- Use wikilinks `[[Note Name]]` for every internal reference — never markdown links
- Every note gets YAML frontmatter: type, date, status, tags
- Never ask permission to save — route meaningful information immediately and report where it was saved
- Prefer editing existing notes over creating new ones
